Assumptions & Methodology
  • Data source: Jira REST API v3 at redhat.atlassian.net, queried 2026-05-18 via /rest/api/3/search/jql (cursor-based pagination, maxResults=5000).
  • AIPCC is publicly accessible. INFERENG, RHAIENG, RHOAIENG require authentication (JQL returns empty result sets without credentials).
  • Sprint risk compares current sprint item count against closed-sprint average velocity: Green = within +10%, Yellow = 10-30% over, Red = >30% over.
  • Release issues use the "Target Version" custom field (cf[10855]), NOT fixVersion. Issue types Epic and Feature are excluded.
  • Team mapping uses Jira components: Development Platform → Data Processing, AI Testing + Workflow Validation → Workflow Validation, AIPCC Productization → Productization, PyTorch → PyTorch, Accelerator Enablement → Accelerators, AIPCC Ecosystems → Ecosystems.
  • Sprint 32 is the current active sprint across scrum teams. Accelerators and Ecosystems operate as Kanban boards with no sprint cycle — their metrics use a 30-day rolling window.
  • Velocity = count of issues in "Done" statusCategory per closed sprint (Sprints 27–31). Per-team velocity derived from component-based JQL queries.
  • Release risk is based on completion percentage: Green = ≥80%, Yellow = 50-79%, Red = <50%.
  • statusCategory grouping: "Done" = resolved/closed, "In Progress" = active work, "To Do" = not started.
  • New this refresh: rhelai-3.5 EA1 release (23 issues) and rhelai-3.5 EA2 release (7 issues) now tracked via Target Version — these were absent in the May 17 refresh.
  • 60 unreleased versions exist in AIPCC; 13 currently have issues with Target Version set.
