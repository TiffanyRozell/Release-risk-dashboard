Assumptions & Methodology ▼

Data Source: Jira REST API v3 at redhat.atlassian.net (unauthenticated access).

Projects Queried: AIPCC, RHELAI, PSAP are publicly accessible. INFERENG, RHAIENG, RHOAIENG were not found — they either require authentication, use different keys, or are on a different Jira instance.

Sprint Velocity: True sprint velocity requires Jira Agile API authentication (board/sprint endpoints). The "velocity" shown here is the 30-day resolved issue count, used as a proxy. Kanban boards use the same metric.

Target Version Field: Issue counts per release use the "Target Version" custom field (customfield_10855) where available, with fixVersion as fallback.

Risk Calculation (Sprint): Green = WIP:Throughput ratio ≤ 1.1x (healthy flow). Yellow = ratio 1.1–1.3x. Red = ratio > 1.3x or throughput near zero.

Risk Calculation (Release): Green = within +10% of average velocity to complete remaining work. Yellow = 10–30% over capacity. Red = >30% over capacity or overdue.

Aging WIP: Issues in "In Progress" status category with last update > 14 days ago.

Limitations: Custom field queries may return incomplete results without authentication. Board/sprint/velocity data from Jira Agile API is not available without auth.
