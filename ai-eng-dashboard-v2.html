Assumptions & Methodology ▼
  • Data source: Jira REST API v3 at redhat.atlassian.net, queried 2026-06-05.
  • Projects accessible: Only AIPCC is publicly accessible without authentication. INFERENG, RHAIENG, and RHOAIENG return no data without Jira credentials.
  • WIP: Issues with statusCategory = "In Progress" (includes statuses: In Progress, Review, Code Review, etc.).
  • 30-day metrics: Resolved = statusCategory=Done AND resolved >= -30d. Created = created >= -30d.
  • Aging WIP: statusCategory = "In Progress" AND updated < -14d (not updated in 14+ days).
  • Velocity: Weekly resolved counts for the last 6 weeks. AIPCC operates as Kanban (no sprint boundaries), so weekly throughput is used.
  • Release data: Uses "Target Version" custom field (customfield_10855), NOT fixVersion. Excludes Epic and Feature issue types.
  • Risk (Release): Red = overdue with remaining work, or <50% done with <14d left. Yellow = <70% done with <30d left. Green = >70% done or >30d remaining.
  • Risk (Sprint): Compares current week velocity to 6-week average. Green = within ±10%. Yellow = 10-30% deviation. Red = >30% deviation.
