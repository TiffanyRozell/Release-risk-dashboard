Assumptions, Data Notes & Risk Logic

Data Source & Scope

  • Data pulled from Jira projects AIPCC, INFERENG, RHAIENG, RHOAIENG via REST API on March 16, 2026.
  • Teams are organized by Jira component — every component with active sprint items is included.
  • Issue types counted: Story, Task, Spike, Bug. Epics and Features excluded from velocity.
  • "Completed" = statusCategory = Done. Includes Closed, Done, Resolved statuses.

Sprint Throughput Logic

  • Scrum teams: velocity = average completed items across last 3-6 closed sprints.
  • Kanban teams (INFERENG): bi-weekly throughput = (30-day completed count / 2) as a velocity proxy.
  • Components with only open sprint data: 90-day completed / 6 used as bi-weekly estimate.
  • Sprint cadences vary: 2 weeks (AIPCC), 2-3 weeks (RHAIENG/RHOAIENG teams).

Risk Thresholds - Sprint

  • Green: Active load within +10% of avg throughput.
  • Yellow: 10-30% above average.
  • Red: >30% above average.
  • Blue: >30% below average (early sprint or items not yet added).

Risk Thresholds - Release

  • Green: >70% complete, OR >30 days to due date.
  • Yellow: <70% done AND <30 days to due date; or <50% done with no date.
  • Red: <50% done AND <14 days to due; or past due with unfinished work.

Target Version Field

  • fixVersion is the active field for version tracking. Target Version (customfield_12319940) is no longer in use.
  • Due dates come from version metadata (releaseDate), not individual issues.
