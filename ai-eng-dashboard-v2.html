Assumptions & Methodology
  • Data source: Jira REST API v3 at redhat.atlassian.net, queried 2026-05-15 via /rest/api/3/search/jql (cursor-based pagination).
  • AIPCC is publicly accessible. INFERENG, RHAIENG, RHOAIENG require authentication (return empty result sets without credentials).
  • Sprint risk compares current sprint item count against closed-sprint average velocity: Green = within +10%, Yellow = 10-30% over, Red = >30% over.
  • Release issues use the "Target Version" custom field (cf[10855]), NOT fixVersion. Issue types Epic and Feature are excluded.
  • Velocity = count of issues in "Done" statusCategory per closed sprint. DP/Prod/PyTorch/Acc: last 5 sprints (S27-S31). WV: last 4 sprints (S28-S31). Accelerators and Midstream Integration boards had no active sprint at query time.
  • Release risk is based on completion percentage: Green = ≥80%, Yellow = 50-79%, Red = <50%.
  • statusCategory grouping: "Done" = resolved/closed, "In Progress" = active work, "To Do" = not started.
  • Boards identified: DP (3719), Workflow Validation (3723), Accelerators (3729), Productization (3731), PyTorch (3735), MidstreamIntegration (10378).
  • Sprint dates extracted from customfield_10020 sprint metadata.
  • The Agile Board API (/rest/agile/1.0/board) requires authentication and was not used. All sprint data was derived from JQL search queries against the sprint custom field.
  • RHAII-3.5 (due 2026-06-18): 10 issues tracked with 0% completion; included for visibility as a future release.
  • 77 unreleased versions exist in AIPCC; only 15 have issues with Target Version set. The remaining 62 are omitted.
