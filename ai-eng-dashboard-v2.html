Assumptions & Methodology
  • Data source: Jira REST API v3 (GET /rest/api/3/search/jql, cursor-based pagination, anonymous access) queried live on 2026-05-01.
  • Projects queried: AIPCC, RHELAI, PSAP, RHAIFIRST. The originally-requested INFERENG, RHAIENG, and RHOAIENG project keys return zero issues on redhat.atlassian.net (empty or restricted access). The closest accessible matches were substituted.
  • Velocity = resolved issues per 2-week window (proxy for sprint velocity since Agile board API requires auth). Computed from 6 trailing 14-day periods for AIPCC; other projects have insufficient throughput for windowed velocity.
  • "Target Version" (customfield_10855) used for release mapping, NOT fixVersion.
  • Issue types excluded: Epic, Feature (per release mapping query).
  • Risk thresholds (Release Risk):
    • Green: remaining ≤ avg_velocity × 1.1 (within +10% of avg velocity)
    • Yellow: remaining ≤ avg_velocity × 1.3 (10-30% over avg velocity)
    • Red: remaining > avg_velocity × 1.3 (>30% over avg velocity)
  • Aging WIP = in-progress issues not updated for >14 days. Currently 308 of 671 WIP items (45.9%) are stale across all projects.
  • Jira API note: Red Hat Jira uses v3 cursor-based search (/rest/api/3/search/jql). Counts obtained via paginated result set length (v3 does not return a total field).
