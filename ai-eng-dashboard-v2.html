Assumptions & Methodology
  • Data source: Jira REST API v3 at redhat.atlassian.net, queried 2026-05-21 via /rest/api/3/search/jql (cursor-based pagination, maxResults=5000).
  • AIPCC is publicly accessible. INFERENG, RHAIENG, RHOAIENG require authentication (JQL returns empty result sets without credentials).
  • Sprint data is live from today's JQL queries using sprint in openSprints(). All 5 boards now on Sprint 33 (May 19 – Jun 2). KPIs (WIP, throughput, aging, created) are also live.
  • Sprint risk compares current sprint item count against closed-sprint average velocity: Green = within +10%, Yellow = 10-30% over, Red = >30% over.
  • Release issues use the "Target Version" custom field (cf[10855]), NOT fixVersion. Issue types Epic and Feature are excluded.
  • Board mapping: Development Platform (board 3719), Workflow Validation (board 3723), AIPCC Accelerators (board 3729), Productization (board 3731), PyTorch (board 3735).
  • Velocity = count of issues in "Done" statusCategory per closed sprint (last 5 sprints). DP avg 71, WV avg 40, Accelerators avg 31, Productization avg 49, PyTorch avg 157.
  • Release risk is based on completion percentage: Green = ≥80%, Yellow = 50-79%, Red = <50%.
  • statusCategory grouping: "Done" = resolved/closed, "In Progress" = active work, "To Do" = not started.
  • Changes from May 20: WIP 600→603 (+3), throughput 1,543→1,513 (-30, rolling window), created 1,683→1,669 (-14), aging WIP 226→227 (+1). All teams now on Sprint 33. Sprint 32 closed: DP=74, WV=49, Acc=2, Prod=47, PT=157.
  • 14 unreleased versions in AIPCC currently have issues with Target Version set. rhoai-3.5.EA2 done 20→21 (+1). RHAII-3.5 EA1 done 22→23 (+1).
