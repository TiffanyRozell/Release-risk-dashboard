Assumptions & Methodology
  • Data source: Jira project AIPCC on redhat.atlassian.net, queried via REST API v3 on 2026-04-10.
  • Target Version field: customfield_10855 (not fixVersion). Issues queried with issuetype not in (Epic, Feature).
  • Velocity: 30-day resolved count (1,201) divided into ~6 weekly buckets. Actual sprint data unavailable (Agile board API requires additional auth).
  • Risk thresholds (Sprint): Green = remaining work within +10% of avg velocity; Yellow = 10-30% over; Red = >30% over capacity.
  • Risk thresholds (Release): Based on % done, days remaining, and ratio of remaining work to historical throughput.
  • Projects INFERENG, RHAIENG, RHOAIENG: Returned 404/empty from API. These require authenticated Jira MCP integration to populate.
  • Versions with 0 issues: RHELAI versions (3.4 EA2, 3.4 GA, 3.5 series) have no issues tagged to Target Version. Work may be tracked differently or not yet planned.
  • Overdue versions: Multiple RHOAI 2.x versions show past release dates but remain unreleased. These may be internal planning artifacts.
