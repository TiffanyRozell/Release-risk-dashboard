Assumptions & Methodology ▼
  • Data source: Jira REST API v3 (search/jql) at redhat.atlassian.net, queried 2026-06-15. Public API access (no authentication).
  • Project mapping: AIPCC → AIPCC, INFERENG → PSAP (Performance and Scale for AI Platforms), RHAIENG → RHELAI (Red Hat Enterprise Linux AI), RHOAIENG → RHODS (Red Hat OpenShift Data Science). The original project keys INFERENG, RHAIENG, and RHOAIENG do not exist in redhat.atlassian.net; closest matches used.
  • Boards/Sprints: Agile board API requires authentication; velocity is approximated from biweekly resolved issue counts over 84 days (6 × 14-day windows).
  • Version queries: Issues linked via "Target Version" (customfield_10855). Only non-Epic, non-Feature issue types counted.
  • Sprint risk badge: Green = WIP:Throughput ratio < 1.1; Yellow = 1.1–4.0 or low activity; Red = > 4.0 or zero throughput with significant WIP present.
  • Release risk: Green = >70% done OR >30 days remaining; Yellow = <70% done AND <30 days remaining; Red = <50% done AND <14 days remaining, or overdue with incomplete items.
  • Aging WIP: Issues with statusCategory = "In Progress" not updated in 14+ days.
  • RHAIENG / RHOAIENG: These projects show very high aging WIP ratios (66/66 and 78/78 respectively) with near-zero recent throughput. This likely indicates work has moved to other Jira projects (e.g., AIPCC) or tracking systems.
