Assumptions & Methodology ▼
  • Data source: redhat.atlassian.net REST API (unauthenticated). AIPCC project data only; INFERENG, RHAIENG, RHOAIENG require authentication and are shown as placeholders.
  • Target Version field: Issue counts use customfield_10855 ("Target Version"), queried via project=AIPCC AND "Target Version"="<ver>" AND issuetype not in (Epic,Feature).
  • Status categories: Done/In Progress/To Do mapped via Jira statusCategory.
  • Velocity: Weekly resolved issue counts (rolling 7-day windows over the past 6 weeks). Current week (W-0) is partial.
  • WIP:Throughput ratio: Current WIP (509) / 30-day resolved (1,439) = 0.35. Values <1.0 are healthy.
  • Risk (Sprint Throughput): Green = current velocity within +10% of 6-week avg (304/wk). Yellow = 10-30% deviation. Red = >30% deviation.
  • Risk (Release): Green = >70% done OR >30d left. Yellow = <70% done AND <30d left. Red = <50% done AND <14d left, or overdue with <90% done.
  • Aging WIP: Issues with statusCategory="In Progress" AND updated <= -14d.
  • Issue counts are approximate (Jira approximate-count API) and may vary by ±5%.
