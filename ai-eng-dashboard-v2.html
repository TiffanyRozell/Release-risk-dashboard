Data as of 2026-04-08
Assumptions & Methodology
  • Data Source: Jira REST API v2 at redhat.atlassian.net, queried 2026-04-08.
  • Version/Release Data (LIVE): Retrieved via /rest/api/2/version/{id}/relatedIssueCounts. Scope (TV) column = total issues with Target Version (customfield_10855).
  • Status Breakdown Limitation: Jira JQL search endpoint returned HTTP 410 (deprecated) without authentication. Done/IP/ToDo breakdowns are estimated from fixVersion resolved counts and version lifecycle stage.
  • Project Mapping: RHAIENG → RHELAI (Jira key); RHOAIENG → RHODS (Jira key). INFERENG returned 404.
  • Sprint Data: Agile board API returned HTTP 401 (auth required). Sprint data carried forward from last accessible query (2026-04-04), with Sprint 29 now closed and Sprint 30 estimated as starting.
  • Velocity: Completion rate = Done/Total per closed sprint. Chart shows last 5 closed sprints.
  • Sprint Risk Thresholds: Green (Low): within +10% of historical avg. Yellow (Medium): 10–30% above or lagging. Red (High): >30% above or completion rate <50%.
  • Release Risk: Based on % done: Green (≥70%), Yellow (50–70%), Red (<50%).
  • Target Version: Read from customfield_10855, NOT fixVersion.
  • RHODS note: Active sprints from 2023, project appears legacy/dormant.
