Assumptions & Methodology \u25BC
  • Data source: Jira REST API v3 at redhat.atlassian.net, queried 2026-06-10.
  • Project mapping: AIPCC queried directly. INFERENG mapped to PSAP (Performance and Scale for AI Platforms). RHAIENG mapped to RHELAI (Red Hat Enterprise Linux AI). RHOAIENG mapped to RHODS (Red Hat OpenShift Data Science). These mappings reflect publicly accessible Jira project keys.
  • Sprint velocity: Approximated from issues completed in closed sprints over the past 90 days, divided into ~6 bi-weekly sprint periods. Board-level sprint data requires Jira authentication.
  • Version issues: Queried using "Target Version" (customfield_10855), excluding Epics and Features. Versions with 0 matching issues are hidden.
  • Risk badges (Sprint): Green = remaining work within +10% of avg velocity. Yellow = 10-30% over. Red = >30% over capacity.
  • Risk badges (Release): Green = >70% done or >30 days remaining. Yellow = <70% done and <30 days. Red = <50% done and <14 days, or overdue.
  • WIP counts: All issues in "In Progress" status category, excluding Epics and Features.
  • Throughput: Issues resolved in the last 30 days. Created count = issues created in last 30 days. AIPCC counts capped at 1000 by API pagination limits.
  • Aging WIP: Issues in "In Progress" status category not updated in 14+ days.
