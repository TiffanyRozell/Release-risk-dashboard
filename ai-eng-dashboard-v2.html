Assumptions & Methodology ▶
  • Data source: Jira REST API v3 at redhat.atlassian.net, queried 2026-05-30.
  • Project access: INFERENG, RHAIENG, and RHOAIENG project keys require Jira authentication and returned no issues. Dashboard uses accessible projects: AIPCC (direct match), RHELAI (closest to RHAIENG), RHODS (closest to RHOAIENG). Configure Jira credentials in the Ambient Integrations page for full access.
  • Throughput: 30-day rolling resolved count (Kanban boards). No sprint-based velocity available without board API auth.
  • Release issues: Queried using "Target Version" (customfield_10855), not fixVersion, filtered to exclude Epic and Feature issue types.
  • Risk (Sprint): Green = WIP:Throughput < 1.5 and aging < 20%; Yellow = ratio 1.5–3.0 or aging 20–50%; Red = ratio > 3.0 or aging > 50%.
  • Risk (Release): Green = >70% done OR >30d remaining; Yellow = <70% done AND <30d remaining; Red = <50% done AND <14d remaining, or overdue.
  • RHELAI & RHODS: Near-zero 30-day throughput suggests these projects may have migrated work to AIPCC or other trackers. Aging WIP items are likely stale.
  • Status categories: Done = Jira statusCategory "Done"; In Progress = "In Progress"; To Do = "To Do".
