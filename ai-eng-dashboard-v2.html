Assumptions & Methodology ▼
  • Data source: Red Hat Jira (redhat.atlassian.net), queried via REST API v3 on 2026-05-09.
  • Auth-gated projects: INFERENG, RHAIENG, and RHOAIENG require Jira authentication and returned zero results via the public API. Their data is shown as unavailable. Only AIPCC data is live.
  • Velocity: Calculated as issues resolved per 2-week window (sprint proxy) over the last 12 weeks. Agile board/sprint APIs require authentication, so this uses JQL-based resolved date ranges.
  • Release issues: Queried using the "Target Version" custom field (cf[10855]), not fixVersion, as the project primarily uses this field for release planning.
  • Risk calculation (Sprint): Green = remaining work within +10% of average velocity; Yellow = 10-30% over; Red = >30% over average velocity.
  • Risk calculation (Release): Green = >70% done or >30 days to release; Yellow = <70% done and <30 days; Red = <50% done and <14 days, or overdue.
  • Issue types: Epics and Features are excluded from release counts to focus on deliverable work items.
  • Status categories: Jira statusCategory grouping: "To Do", "In Progress", "Done".
