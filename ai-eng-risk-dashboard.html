Assumptions & Methodology
  • Data source: Jira REST API v3 (anonymous access)
  • INFERENG, RHAIENG, RHOAIENG require authenticated access; data shown only for AIPCC.
  • Velocity = resolved issues per 2-week window (proxy for sprint velocity since Agile board API requires auth).
  • "Target Version" (customfield_10855) used for release mapping.
  • Aging WIP = in-progress issues not updated for >14 days.
  • Risk thresholds (Release Burndown):
    • Green: >70% done OR >30 days left
    • Yellow: <70% done AND <30 days left
    • Red: <50% done AND <14 days left, or overdue
    • No release date: >70% = Green, 50-70% = Yellow, <50% = Red
  • Flow health: WIP:Throughput < 0.5 = Healthy, 0.5-1.0 = Warning, >1.0 = Critical
