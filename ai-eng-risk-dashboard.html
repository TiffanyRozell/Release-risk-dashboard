Assumptions & Methodology ▼

Data Source: Jira REST API v3 at redhat.atlassian.net (unauthenticated). Queried on 2026-05-26.

Release Burndown Risk: Green = >70% done OR >30 days remaining. Yellow = <70% done AND <30 days remaining. Red = <50% done AND <14 days remaining, or already overdue with incomplete work.

Aging WIP: Issues with statusCategory "In Progress" and updated date >14 days ago. "Days Stale" is calendar days since last update. Top 20 oldest issues shown per project.

Flow Metrics — WIP:Throughput Ratio: Current WIP divided by 30-day throughput. Values >1.0 indicate WIP accumulating faster than completion. Healthy range is 0.3–0.5x.

Flow Metrics — Created vs Resolved: 30-day created vs 30-day resolved issue counts. A ratio >1.0 means more issues arrive than are completed (backlog growing).

Limitations: INFERENG, RHAIENG, RHOAIENG not accessible. Sprint/board data requires Agile API auth. Custom field "Target Version" (cf[10855]) used for version assignments; results may be incomplete without auth. Velocity proxy uses 30-day resolved count.
