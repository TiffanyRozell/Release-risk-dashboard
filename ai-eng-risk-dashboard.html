Assumptions & Methodology
  • Data source: Jira REST API v3 at redhat.atlassian.net, queried 2026-05-18 via /rest/api/3/search/jql (cursor-based pagination, maxResults=5000). Only AIPCC project accessible without authentication.
  • INFERENG, RHAIENG, RHOAIENG require Jira authentication. JQL queries return empty result sets without credentials. Configure credentials via Workspace Settings > Integrations to include their data.
  • Release burndown risk: Green = ≥70% done or >30 days remaining, Yellow = <70% done and <30 days remaining, Red = <50% done and <14 days remaining, or overdue.
  • Release issues queried via "Target Version" (cf[10855]), NOT fixVersion. Epic and Feature issue types excluded.
  • Aging WIP: Issues with statusCategory = "In Progress" and updated date >14 days ago. 236 of 631 total WIP items (37%) are stale — consistent with 230/628 (37%) on 2026-05-17.
  • WIP:Throughput ratio = current WIP count / 30-day resolved count. Values <1.0 generally healthy; >2.0 indicates flow congestion.
  • Net flow: 30-day created (1,564) minus 30-day resolved (1,442) = +122 net growth. Improved from +144 on prior refresh.
  • Known release dates: RHAII-3.5 EA1 = 2026-06-02, rhoai-3.5.EA1 = 2026-06-02, RHAII-3.5 = 2026-06-17, rhelai-3.5 = 2026-06-18, rhelai-3.5 EA1 release = 2026-06-18, RHAII-3.5 EA2/rhoai-3.5.EA2 = 2026-06-30, rhoai-3.5 = 2026-07-14, rhelai-3.5 EA2 release = 2026-07-16, RHAII-3.5 GA = 2026-07-30, rhelai-3.5 GA = 2026-08-20.
  • New this refresh: rhelai-3.5 EA1 release (23 issues, 48% done, due Jun 18) and rhelai-3.5 EA2 release (7 issues, 57% done, due Jul 16) now tracked — these versions were not present in the May 17 refresh. RHELAI filter added to burndown.
  • statusCategory grouping: "Done" = resolved/closed, "In Progress" = active work, "To Do" = not started.
  • The Agile Board API requires authentication and was not used. All data derived from JQL search queries.
