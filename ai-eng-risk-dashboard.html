Assumptions & Methodology
  • Data source: Jira REST API v3 (GET /rest/api/3/search/jql, cursor-based pagination, anonymous access) queried live on 2026-05-03.
  • Projects: AIPCC, RHELAI, PSAP, RHAIFIRST. Originally-requested INFERENG, RHAIENG, RHOAIENG return zero issues on redhat.atlassian.net (empty or restricted access); closest accessible matches were substituted.
  • "Target Version" (customfield_10855) used for release mapping, NOT fixVersion.
  • Issue types excluded: Epic, Feature (per release mapping query).
  • Aging WIP = in-progress issues not updated for >14 days. Total: 315 of 711 WIP items (44.3%) across all projects.
  • Risk thresholds (Release Burndown):
    • Green: >70% done OR >30 days left with reasonable progress
    • Yellow: <70% done AND <30 days left
    • Red: <50% done AND <14 days left, or overdue
    • No release date: >70% = Green, 50-70% = Yellow, <50% = Red
  • Flow health (WIP:Throughput ratio): <0.5 = Healthy, 0.5–1.0 = Warning, >1.0 = Critical
  • Velocity proxy: Resolved issues per 2-week window derived from 90-day resolved counts. Agile board/sprint APIs require authentication.
  • RHELAI note: 100% of WIP is stale. Zero issues created or resolved in 30 days. Project appears to be in transition or winding down.
