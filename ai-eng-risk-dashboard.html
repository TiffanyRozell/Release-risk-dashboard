Assumptions & Methodology ▶
  • Data source: Jira REST API (search/jql endpoint) at redhat.atlassian.net, queried 2026-06-04.
  • Project access: INFERENG, RHAIENG, and RHOAIENG project keys require Jira authentication and returned empty results via anonymous API access. Dashboard uses accessible projects: AIPCC (direct match), RHELAI (proxy for RHAIENG), RHODS (proxy for RHOAIENG). To include all requested projects, configure Jira credentials on the Ambient Integrations page.
  • Release Burndown risk: Green = >70% done OR >30d remaining; Yellow = <70% done AND <30d remaining; Red = <50% done AND <14d remaining, or overdue.
  • Release issues: Queried using "Target Version" (customfield_10855), not fixVersion, filtered to exclude Epic and Feature issue types.
  • Aging WIP: Issues in Jira statusCategory "In Progress" with last update >14 days ago. Top 20 per project shown, sorted by stalest first.
  • Flow Metrics: WIP = current In Progress count; Throughput = issues resolved in last 30 days; Created = issues created in last 30 days. WIP:Throughput ratio <1.5 = Healthy, 1.5–3.0 = Warning, >3.0 = Critical.
  • RHELAI & RHODS: Near-zero throughput and high aging percentages suggest these trackers are legacy. Active development likely tracked in AIPCC or authenticated-only projects (RHOAIENG, RHAIENG, INFERENG).
  • Days stale: Calculated from each issue's "updated" timestamp to 2026-06-04.
