Assumptions & Methodology \u25BC
  • Data source: Jira REST API v3 at redhat.atlassian.net, queried 2026-06-10.
  • Project mapping: AIPCC queried directly. INFERENG → PSAP (Performance and Scale for AI Platforms). RHAIENG → RHELAI (Red Hat Enterprise Linux AI). RHOAIENG → RHODS (Red Hat OpenShift Data Science).
  • Release risk: Green = >70% done or >30 days remaining. Yellow = <70% done and <30 days remaining. Red = <50% done and <14 days remaining, or past release date.
  • Aging WIP: Issues with statusCategory = "In Progress" and last updated >14 days ago. Top 20 shown per project.
  • Flow metrics: WIP = all issues in "In Progress" status category. Throughput = issues resolved in last 30 days. Created = issues created in last 30 days. Net flow = created − resolved (negative = draining backlog).
  • WIP:Throughput ratio: Lower is better. Under 2.0 indicates healthy flow; above 5.0 suggests bottlenecks. "Stalled" indicates WIP exists but no throughput.
  • API limitations: AIPCC throughput/created counts capped at 1,000 by pagination. Board and sprint APIs require authentication.
  • Versions: Queried using "Target Version" (customfield_10855), excluding Epics and Features. Only versions with ≥1 matching issue are shown.
