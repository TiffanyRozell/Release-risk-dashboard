Assumptions & Methodology ▶
  • Data source: Jira REST API v3 (unauthenticated) at redhat.atlassian.net, queried 2026-05-27.
  • Projects queried: AIPCC (AI Platform Core Components), RHELAI (Red Hat Enterprise Linux AI), RHODS (Red Hat OpenShift Data Science). The requested keys INFERENG, RHAIENG, and RHOAIENG do not exist; RHELAI and RHODS are the closest matches.
  • Version issues: Queried using "Target Version" (customfield_10855), not fixVersion.
  • Sprint data: Board/sprint APIs require authentication. Using 30-day Kanban throughput instead.
  • Release risk calculation:
    • Green: ≥70% done OR >30 days until release
    • Yellow: <70% done AND <30 days remaining
    • Red: <50% done AND <14 days remaining, or overdue
  • Aging WIP: Issues with statusCategory = "In Progress" and updated < -14d.
  • Flow health: WIP:Throughput ratio <1 is healthy; >3 indicates flow problems; >10 indicates stalled work.
  • RHELAI & RHODS: These projects show near-zero recent activity (0 created, 1 resolved in 30 days) with high aging WIP. This likely indicates project tracker migration rather than actual stalled work. The RHOAIENG project key referenced by the user may exist in a different permission scope.
