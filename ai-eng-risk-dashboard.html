Assumptions & Methodology ▼
  • Data source: redhat.atlassian.net REST API v3 (unauthenticated, approximate-count endpoint). AIPCC project data only; INFERENG, RHAIENG, RHOAIENG require authentication.
  • Target Version field: Release burndown counts use customfield_10855 ("Target Version"), queried via project=AIPCC AND "Target Version"="<ver>" AND issuetype not in (Epic,Feature).
  • Release Risk (Burndown tab): Green = >70% done OR >30d left. Yellow = <70% done AND <30d left. Red = <50% done AND <14d left, or overdue with <90% done. Undated versions with <70% done = Yellow.
  • Aging WIP: Issues with statusCategory="In Progress" AND updated <= -14d. Stale days calculated from last updated date to 2026-06-06.
  • WIP:Throughput ratio: Current WIP / 30-day resolved. Values <1.0 indicate healthy flow; >2.0 signals bottleneck risk.
  • Net Inflow: 30d created minus 30d resolved. Positive values indicate growing backlog.
  • Weekly created counts in the chart are estimates based on the 30-day total (1,553) distributed proportionally to resolved patterns. Exact weekly created counts require authenticated JQL access.
  • Issue counts are approximate (Jira approximate-count API) and may vary by ±5%.
