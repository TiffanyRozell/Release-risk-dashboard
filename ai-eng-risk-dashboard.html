Assumptions & Methodology ▼
  • Data source: Jira REST API v3 at redhat.atlassian.net, queried 2026-06-05.
  • Projects accessible: Only AIPCC is publicly accessible. INFERENG, RHAIENG, and RHOAIENG require Jira authentication and returned no data.
  • WIP: Issues with statusCategory = "In Progress" (includes statuses: In Progress, Review, Code Review, Development, etc.).
  • Aging WIP: statusCategory = "In Progress" AND updated < -14d. Days stale calculated from last update date to 2026-06-05.
  • Throughput: 30-day resolved = statusCategory=Done AND resolved >= -30d.
  • Created: Issues with created >= -30d regardless of status.
  • WIP:Throughput Ratio: Current WIP / 30-day throughput. Lower is better. <1.0 = healthy, 1.0-2.0 = caution, >2.0 = critical.
  • Release data: Uses "Target Version" custom field (customfield_10855), NOT fixVersion. Excludes Epic and Feature issue types.
  • Release Risk: Red = overdue with remaining work, or <50% done with <14d left. Yellow = <70% done with <30d left. Green = >70% done or >30d remaining.
  • Severity sort: 1 (Red) > 2 (Yellow) > 3 (Green). Within same severity, sorted by days remaining ascending.
