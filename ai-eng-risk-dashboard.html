Assumptions & Methodology ▼
  • Data source: Red Hat Jira (redhat.atlassian.net), queried via REST API v3 on 2026-05-09.
  • Auth-gated projects: INFERENG, RHAIENG, and RHOAIENG require Jira authentication and returned zero results via the public API. Only AIPCC data is live.
  • Release Burndown risk: Green = >70% done or >30 days to release. Yellow = <70% done and <30 days left. Red = <50% done and <14 days left, or past due date.
  • Aging WIP: Issues in "In Progress" statusCategory with last update >14 days ago. Sorted oldest-first.
  • Flow Metrics: WIP:Throughput ratio = current in-progress count / 30-day resolved count. A ratio >1.0 means work is accumulating. Created vs Resolved imbalance indicates backlog growth or contraction.
  • Target Version: Uses custom field cf[10855] ("Target Version"), not fixVersion.
  • Issue types: Epics and Features excluded from release counts.
  • Staleness: Calculated as days since last Jira update, not days since status change.
