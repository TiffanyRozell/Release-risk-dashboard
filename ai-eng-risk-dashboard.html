Assumptions & Methodology ▼
  • Data source: Jira REST API v3 (search/jql) at redhat.atlassian.net, queried 2026-06-11. Public API access (no authentication).
  • Project mapping: AIPCC → AIPCC, INFERENG → PSAP (Performance and Scale for AI Platforms), RHAIENG → RHELAI (Red Hat Enterprise Linux AI), RHOAIENG → RHODS (Red Hat OpenShift Data Science).
  • Release burndown risk: Green = >70% done OR >30 days remaining; Yellow = <70% done AND <30 days left; Red = <50% done AND <14 days left, or overdue with incomplete items.
  • Aging WIP: Issues with statusCategory = "In Progress" and updated < -14d. Stale days calculated from last update to 2026-06-11. Top 20 items per project shown, sorted oldest first.
  • Flow health: Green = WIP:Throughput < 1.5 and aging < 30%; Yellow = low volume or aging 30-75%; Red = zero throughput with WIP present, or >75% aging.
  • Version queries: Issues linked via "Target Version" (customfield_10855) first, falling back to fixVersion. Non-Epic, non-Feature types only.
  • Stale projects: RHAIENG and RHOAIENG show patterns consistent with work having moved to other Jira projects. RHOAIENG (RHODS) contains onboarding tasks from 2022-2023 that were never closed.
  • Boards/Sprints: Agile board API requires authentication and was not accessible. No sprint-level data available.
