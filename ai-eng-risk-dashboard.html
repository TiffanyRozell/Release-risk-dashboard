Assumptions & Methodology
  • Data source: Jira REST API v3 at redhat.atlassian.net, queried 2026-05-20 via /rest/api/3/search/jql (cursor-based pagination, maxResults=5000). Only AIPCC project accessible without authentication.
  • INFERENG, RHAIENG, RHOAIENG require Jira authentication. JQL queries return empty result sets without credentials. Configure credentials via Workspace Settings > Integrations to include their data.
  • Release burndown risk: Green = ≥70% done or >30 days remaining, Yellow = <70% done and <30 days remaining, Red = <50% done and <14 days remaining, or overdue.
  • Release issues queried via "Target Version" (cf[10855]), NOT fixVersion. Epic and Feature issue types excluded.
  • Aging WIP: Issues with statusCategory = "In Progress" and updated date >14 days ago. 226 of 600 total WIP items (38%) are stale — improved from 258/591 (44%) on 2026-05-19.
  • WIP:Throughput ratio = current WIP count / 30-day resolved count. Values <1.0 generally healthy; >2.0 indicates flow congestion. Current: 0.39 (stable).
  • Net flow: 30-day created (1,683) minus 30-day resolved (1,543) = +140 net growth. Worsened from +102 on prior refresh due to higher creation rate.
  • Known release dates: RHAII-3.5 EA1 = 2026-06-02, rhoai-3.5.EA1 = 2026-06-02, RHAII-3.5 = 2026-06-17, rhelai-3.5 EA1 release = 2026-06-18, RHAII-3.5 EA2/rhoai-3.5.EA2 = 2026-06-30, rhoai-3.5 = 2026-07-14, rhelai-3.5 EA2 release = 2026-07-16, RHAII-3.5 GA = 2026-07-30, rhelai-3.5 GA = 2026-08-20.
  • Changes from May 19: WIP 591→600 (+9), throughput 1,534→1,543 (+9), created 1,636→1,683 (+47), aging WIP 258→226 (-32, improved), aging ratio 44%→38%. Net flow worsened +102→+140. rhoai-3.5.EA1 done 66→69, rhoai-3.5.EA2 total 38→50 (12 issues added), RHAII-3.5 EA1 total 39→42, rhoai-3.4 total 37→36.
  • statusCategory grouping: "Done" = resolved/closed, "In Progress" = active work, "To Do" = not started.
  • The Agile Board API requires authentication and was not used. All data derived from JQL search queries.
